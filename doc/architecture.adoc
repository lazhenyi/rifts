== Four-Layer Model

Rift/1 splits the realвҖ‘time stack into four layers (spec В§3):

[horizontal]
Transport::  Carries bidirectional byteвҖ‘streams or datagrams (WebSocket, WebTransport, TCP).
Session::    Identity, connection lifetime, recovery, heartbeat, flowвҖ‘control negotiation.
Message::    Message type, reliability, ordering, ack, replay, priority, TTL.
Topic::       Subscription, publish, permission, sharding, fanout, retention, snapshot.

Each layer is represented by a public module in the crate:

[source]
----
src/transport/   вҶ’ Transport abstraction + framework adapters
src/session/     вҶ’ Session lifecycle, auth, resume, offset tracking
src/message/     вҶ’ Event, Command, Reply, State, Datagram, Stream, Snapshot
src/topic/       вҶ’ Topic profiles, store, retention, ordering
src/broker/      вҶ’ InвҖ‘memory broker, router, fanout, dedupe, offsets, snapshots
----

=== Handshake Flow in `Connection::handshake()`

1. `read_frame()` вҖ” expect a `Control`вҖ‘type `hello` frame.
2. Decode `Hello` payload вҖ” extract protocol, version, codecs, auth_modes, token, `last_offsets`.
3. `negotiate_major(client_major)` вҖ” accept only major version 1.
4. `negotiate(server_codecs, client_codecs)` вҖ” pick JSON or CBOR.
5. `auth_provider.authenticate(mode, token)` вҖ” produce `AuthContext { client_id, claims }`.
6. `ResumeManager::evaluate(...)` вҖ” decide `FullResume | PartialResume | SnapshotRequired | ColdStart`.
7. Write `welcome` + `ready` frames synchronously (before spawning reader/writer tasks).

== Transport Architecture

The `Transport` / `TransportListener` / `TransportConnection` traits are the abstraction
boundary.  Each transport binding provides independent read / write halves so that the
serverвҖҷs reader and writer tasks never contend on a mutex.

*Standalone WebSocket* (`features = ["websocket"]`)::
  `tokio-tungstenite` is split with `WebSocketStream::split()` at accept time.
  The reader half is used by the reader task; the writer half is wrapped in
  `Arc<tokio::sync::Mutex<SplitSink>>` and shared between the handshake writer
  and the writer task.

*Framework adapters* (`features = ["axum"]`, etc.)::
  Each adapter implements `TransportConnection` for the frameworkвҖҷs WebSocket type
  (axum uses `Arc<tokio::sync::Mutex<WebSocket>>`; actix and ntex use a channel bridge
  because their WS types are `!Send`).

== Error Model

Errors use `thiserror` deeply.  The topвҖ‘level `RiftError` composes six typed
subвҖ‘enums:

[source,rust]
----
pub enum RiftError {
    Frame(FrameReject),       // version, codec, schema, encoding
    Session(SessionReject),   // not found, expired, conflict, resume
    Topic(TopicReject),       // not found, closed, rate-limited, forbidden
    Message(MessageReject),   // duplicate, expired, too large, ack timeout
    Auth(AuthReject),         // required, invalid, expired, revoked, denied
    System(SystemReject),     // overloaded, maintenance, shard-moved, internal
    Config(ConfigError),      // invalid configuration
    Io(std::io::Error),
}
----

Large error sources (tungstenite, ciborium, serde_json, etc.) are boxed inside
a `BoxedStdError` wrapper so the `Result` type stays small on the stack.

== Backpressure

`BackpressureController` guards the perвҖ‘connection send queue with an atomic
CASвҖ‘loop enqueue.  When the queue exceeds the highвҖ‘water mark (90% of `max_send_queue_bytes`),
the controller selects a strategy:

[cols="2,5a",options="header"]
|===
| Strategy | Behaviour

| `Pause`
| Caller spins with `tokio::time::sleep` until capacity frees.

| `DropVolatile`
| Messages with `Priority::Volatile` or `Priority::Background` are discarded.

| `CoalesceState`
| State messages for the same `state_key` are collapsed to the latest.

| `Downgrade`
| All messages are deferred; the writer is throttled.

| `Disconnect`
| The slow consumer is kicked with `CloseCode::SlowConsumer`.

| `SnapshotLater`
| The connection switches to periodic snapshot polling instead of live messages.
|===
