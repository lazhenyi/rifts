The `rifts` crate is the canonical Rust server implementation of the **Rift Realtime Protocol (Rift/1)** —
a bidirectional messaging protocol for real‑time applications.  Rift/1 replaces raw WebSocket +
event‑string patterns with explicit semantics: eight message classes, offset‑based resume,
topic‑level QoS, schema‑first events, and protocol‑level backpressure.

== Crate Features

[cols="3m,7a",options="header"]
|===
| Feature | Description

| `websocket` *(default)*
| Standalone WebSocket transport (tokio‑tungstenite).  No framework dependency.

| `axum`
| Accept Rift/1 connections from `axum::extract::ws::WebSocket`.

| `actix-web`
| Accept Rift/1 connections from actix‑web WebSocket sessions.

| `warp`
| Accept Rift/1 connections from `warp::ws::WebSocket`.

| `ntex`
| Accept Rift/1 connections from ntex WebSocket handlers.

| `tokio-frameworks`
| Shortcut: enables `axum` + `warp`.

| `all-frameworks`
| Shortcut: enables `axum` + `actix-web` + `warp` + `ntex`.
|===

== Quick Install

[source,sh]

[source,toml]
----
